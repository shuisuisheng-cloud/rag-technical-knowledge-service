# LeetCode Day 9 反转字符串

## 题目目标

输入是什么？

第一行输入字符串长度 `n`，第二行输入需要反转的字符串。

输出是什么？

输出反转后的字符串。

题目禁止使用哪些 Python 内置方法？

- 字符串切片反转 `s[::-1]`
- `reversed()`
- `list.reverse()`

题目要求通过双指针和字符交换完成反转。

## 为什么使用双指针

为什么一个指针从左侧开始，另一个从右侧开始？

因为字符串反转的本质是让最左侧字符与最右侧字符交换，然后让第二个字符与倒数第二个字符交换，依次向中间处理。

因此设置：

```python
left = 0
right = n - 1
```

每次交换后两个指针怎样移动？

```python
left += 1
right -= 1
```

左指针向右移动，右指针向左移动，继续处理下一对字符。

什么时候结束循环？

当：

```python
left >= right
```

时结束。

代码中使用：

```python
while left < right:
```

保证只有左右指针尚未相遇时才继续交换。

## 核心变量

- `left`：左指针，指向当前需要交换的左侧字符。
- `right`：右指针，指向当前需要交换的右侧字符。
- `chars`：由原字符串转换得到的可修改字符列表。
- `n`：输入字符串的长度。

`left` 初始为 `0`。

`right` 初始为 `n - 1`，因为长度为 `n` 的字符串最后一个合法下标是 `n - 1`。

`chars` 用来保存可修改的字符序列，因为 Python 字符串本身不能直接修改。

## 为什么 Python 字符串不能直接原地修改

Python 的字符串是不可变对象。

字符串创建以后，不能通过下标直接修改其中的字符，例如：

```python
text[0] = "a"
```

会产生错误。

因此需要先转换为列表：

```python
chars = list(text)
```

列表是可变对象，可以通过下标交换元素。

交换完成后，再使用：

```python
result = "".join(chars)
```

将字符列表重新连接成字符串。

## 完整数据流与控制流

读取字符串长度 `n`
→ 读取原始字符串
→ 使用 `list()` 转换为字符列表
→ 初始化 `left = 0`
→ 初始化 `right = n - 1`
→ 判断 `left < right`
→ 交换 `chars[left]` 和 `chars[right]`
→ `left` 向右移动
→ `right` 向左移动
→ 重复判断与交换
→ 两个指针相遇或交错
→ 使用 `join()` 重新组成字符串
→ 输出反转结果

## 交换操作

实际使用的交换语句是：

```python
chars[left], chars[right] = chars[right], chars[left]
```

交换前：

```text
chars[left]  保存左侧字符
chars[right] 保存右侧字符
```

交换后：

```text
chars[left]  保存原来的右侧字符
chars[right] 保存原来的左侧字符
```

Python 的多变量赋值可以直接完成交换，不需要额外定义临时变量。

## 为什么循环条件是 left < right

使用：

```python
while left < right:
```

表示只有左指针仍然位于右指针左侧时，才需要继续交换。

如果字符串长度为奇数，两个指针最终会指向中间的同一个字符。

中间字符不需要与自己交换，因此当：

```python
left == right
```

时可以直接结束。

如果写成：

```python
while left <= right:
```

当两个指针相等时，会额外执行一次中间字符与自身的交换。

结果通常不会改变，但这次操作没有意义。

## 输入与输出

输入格式：

```text
第一行：整数 n
第二行：字符串
```

例如：

```text
5
hello
```

输出格式：

```text
输出反转后的字符串
```

例如：

```text
olleh
```

Python ACM 输入可以写成：

```python
import sys

n = int(sys.stdin.readline())
text = sys.stdin.readline().strip()
```

## 时间复杂度

时间复杂度为：

```text
O(n)
```

双指针大约交换 `n / 2` 次。

虽然只处理大约一半次数，但时间复杂度会忽略常数：

```text
O(n / 2) = O(n)
```

此外：

```python
list(text)
```

需要遍历整个字符串，时间复杂度为 `O(n)`。

```python
"".join(chars)
```

也需要遍历整个字符列表，时间复杂度为 `O(n)`。

因此完整程序的时间复杂度仍然是：

```text
O(n)
```

## 空间复杂度

双指针交换过程本身只使用：

- `left`
- `right`

因此算法核心的额外空间复杂度是：

```text
O(1)
```

但是 Python 字符串不可变，需要创建字符列表：

```python
chars = list(text)
```

字符列表需要保存 `n` 个字符，因此需要：

```text
O(n)
```

空间。

所以应区分：

```text
双指针交换过程的额外空间：O(1)
Python 完整实现的额外空间：O(n)
```

## 今日易错点

### `right` 为什么初始化为 `n - 1`

因为长度为 `n` 的列表下标范围是：

```text
0 到 n - 1
```

如果写成：

```python
right = n
```

访问 `chars[right]` 时会发生下标越界。

### 为什么交换后必须同时移动两个指针

当前左右位置交换完成以后，这两个字符已经位于最终位置。

因此必须继续处理更靠近中间的字符：

```python
left += 1
right -= 1
```

如果不移动指针，循环会一直重复交换同一对字符。

如果只移动一个指针，可能破坏已经完成的交换关系。

### `join()` 的作用是什么

`join()` 将字符列表连接成一个完整字符串。

例如：

```python
chars = ["o", "l", "l", "e", "h"]
result = "".join(chars)
```

结果是：

```text
olleh
```

空字符串 `""` 表示字符之间不添加分隔符。

### 为什么不能使用切片反转

下面的写法虽然能够反转字符串：

```python
text[::-1]
```

但题目希望训练：

- 双指针；
- 数组下标；
- 原地交换思想；
- 循环边界；
- 指针移动。

直接使用切片无法体现这些算法过程。

### 输入的 n 与字符串实际长度不一致时怎样处理

在标准算法题中，通常默认输入合法，`n` 与字符串真实长度一致。

为了避免下标越界，也可以直接使用：

```python
right = len(chars) - 1
```

但在 ACM 题目中，应优先按照题目输入格式完成算法，不需要进行过度异常处理。

## 题型识别方法

看到以下关键词时，可以考虑左右双指针：

- 反转字符串；
- 反转数组；
- 首尾交换；
- 从两端向中间处理；
- 回文判断；
- 有序数组寻找目标和；
- 原地修改数组。

左右双指针的典型结构是：

```text
左指针从开头出发
右指针从结尾出发
每轮同时向中间移动
相遇或交错时结束
```

双指针还可以用于：

- 判断回文字符串；
- 移动零；
- 删除有序数组中的重复项；
- 有序数组两数之和；

## 面试时怎么讲

这道题要求不能使用字符串切片和内置反转函数，因此我使用左右双指针。

左指针从下标 `0` 开始，右指针从下标 `n - 1` 开始。

当 `left < right` 时，交换两个指针指向的字符，然后左指针向右移动，右指针向左移动。

当两个指针相遇或交错时，说明所有字符已经完成反转。

由于 Python 字符串不可变，我先把字符串转换为字符列表，交换完成后再使用 `join()` 转回字符串。

双指针交换过程的额外空间是 `O(1)`，但 Python 创建字符列表需要 `O(n)` 空间。

整体时间复杂度为 `O(n)`。

## 如果从零手撕，第一步做什么

第一步先将字符串转换成可修改列表：

```python
chars = list(text)
```

然后初始化左右指针：

```python
left = 0
right = len(chars) - 1
```

写出循环条件：

```python
while left < right:
```

循环中交换字符：

```python
chars[left], chars[right] = chars[right], chars[left]
```

移动指针：

```python
left += 1
right -= 1
```

最后转换回字符串：

```python
result = "".join(chars)
```

边界测试顺序：

1. 单字符；
2. 两个字符；
3. 奇数长度；
4. 偶数长度；
5. 回文字符串。
