# STM32F407VET6 V2 Day 7 DHT11 40bit 数据读取与串口输出

## 今日目标

在 Day 6 已完成 DHT11 启动信号和响应检测的基础上，实现：

- 单个数据位读取；
- 40 bit 数据接收；
- 5 字节数据保存；
- 校验和验证；
- 温度解析；
- USART2 输出网关可解析的数据。

## DHT11 数据结构

DHT11 一次发送 40 bit，共 5 字节：

```text
data[0]：湿度整数部分
data[1]：湿度小数部分
data[2]：温度整数部分
data[3]：温度小数部分
data[4]：校验和
```

校验规则：

```text
data[4]
=
data[0] + data[1] + data[2] + data[3] 的低 8 bit
```

只有校验通过，数据才可以继续解析和发送。

## 单个数据位读取

DHT11 每个数据位先输出一段低电平，再输出一段高电平。

高电平持续时间不同：

```text
较短高电平
→ bit 0

较长高电平
→ bit 1
```

程序在进入高电平后等待约 40 μs，再读取 DATA 引脚：

```text
40 μs 后已经变低
→ bit 0

40 μs 后仍然为高
→ bit 1
```

## DHT11_ReadBit()

`DHT11_ReadBit()` 使用：

```c
int8_t
```

作为返回类型，用于表达三种结果：

```text
-1：读取失败或超时
 0：数据位 0
 1：数据位 1
```

使用有符号类型，是因为函数需要返回 `-1` 表示失败。

## DHT11_ReadData()

`DHT11_ReadData()` 负责读取完整的一帧 DHT11 数据。

流程：

```text
发送启动信号
→ 等待 DHT11 响应
→ 清空 data[5]
→ 读取 5 个字节
→ 每个字节读取 8 bit
→ 将 bit 依次组装到字节中
→ 计算并验证校验和
→ 返回成功或失败
```

该函数使用：

```c
uint8_t
```

表示：

```text
0：读取失败
1：读取成功
```

## 字节组装

每读取一个 bit，当前字节先左移一位：

```c
data[byte_index] <<= 1;
```

再写入新读取的 bit：

```c
data[byte_index] |= bit;
```

连续执行 8 次后，组成一个完整字节。

总共读取：

```text
5 字节 × 8 bit = 40 bit
```

## 超时保护

所有等待电平变化的过程都必须设置超时。

如果 DHT11 未连接或通信异常，而程序一直等待目标电平，主循环可能永久卡死。

正确流程：

```text
等待电平变化
→ 超时前检测成功：继续读取
→ 达到超时：立即返回失败
```

## 校验和

读取 5 个字节后计算：

```c
uint8_t checksum =
    data[0] +
    data[1] +
    data[2] +
    data[3];
```

然后比较：

```c
checksum == data[4]
```

校验失败可能表示：

- 某个 bit 判断错误；
- 通信时序不稳定；
- 接线或电气干扰；
- 数据读取不完整。

校验失败的数据不能发送给网关。

## 温度解析

温度数据来自：

```text
data[2]
data[3]
```

解析后可以得到带小数的温度，例如：

```text
26.5
```

## USART2 输出格式

为了与 Linux 网关当前解析器对齐，STM32 输出：

```text
temperature:26.5
```

完整串口行以换行结束：

```text
temperature:26.5\r\n
```

网关每次读取一行，并解析冒号后的浮点数。

## 串口参数对齐

STM32 USART2 当前波特率为：

```text
115200
```

Linux 网关真实串口配置也必须改为：

```text
115200
```

如果网关仍使用 9600，可能出现：

- 串口乱码；
- 无法完整读取一行；
- 数据解析失败。

## 完整数据流

```text
DHT11 真实温湿度
→ GPIO 单总线时序
→ 读取 40 bit
→ 保存到 data[5]
→ 校验和验证
→ 温度解析
→ USART2 输出 temperature:<float>
→ Linux 网关读取
→ MQTT 发布
```

## 面试时怎么讲

我在 STM32F407VET6 上使用 GPIO 单总线和 TIM6 微秒级时序完成了 DHT11 数据读取。

程序先发送启动信号并等待响应，再通过高电平持续时间区分 bit 0 和 bit 1，连续读取 40 bit 到 5 字节数组。

`DHT11_ReadBit()` 使用 `int8_t` 返回 `-1、0、1`，分别表示失败、bit 0 和 bit 1。

读取完成后，对前四个字节求和并与第五个字节比较，只有校验通过才解析温度。

最终通过 USART2 输出 `temperature:26.5`，并将波特率和文本格式与 Linux 网关解析器对齐。
