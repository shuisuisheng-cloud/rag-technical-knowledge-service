# LeetCode Day 5 两数之和

## 题目类型

数组、哈希表。

## 题目目标

在数组中寻找两个元素，使它们的和等于目标值，并输出这两个元素的下标。

## 核心思路

遍历数组时，计算当前数字所需要的补数。

使用字典保存已经遍历过的数字及其下标，字典结构为：

`数字 → 下标`

如果补数已经存在于字典中，就可以直接得到两个下标，不需要再次扫描数组。

## 数据流

1. 读取数组长度和目标值；
2. 读取整数数组；
3. 从左到右遍历数组；
4. 计算当前数字对应的补数；
5. 检查补数是否已经存在于字典；
6. 如果存在，输出两个下标；
7. 如果不存在，将当前数字和下标存入字典。

## 今日出现的问题

* 输入内容最初没有全部转换为整数；
* 字典的键和值关系需要明确为“数字 → 下标”；
* 找到补数后应输出补数对应的旧下标和当前下标；
* 找到答案后需要结束当前流程；
* 不应使用 `list.index()` 重复扫描数组。

## 复杂度

时间复杂度：`O(n)`

每个元素只被遍历一次，字典查询的平均时间复杂度为 `O(1)`。

空间复杂度：`O(n)`

最坏情况下，需要把大部分元素存入字典。
