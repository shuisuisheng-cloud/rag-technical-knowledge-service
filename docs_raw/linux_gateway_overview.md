# Linux-STM32 物联网边缘网关

## 项目定位
本项目是 EdgeAIoT 边缘智能系统中的通信层与数据中枢，负责连接 STM32 环境感知与执行控制终端、MQTT Broker 和 Orange Pi 总控层。
网关接收 STM32 通过串口上传的设备数据，完成协议解析、数据处理和 MQTT 转发；同时接收上层控制消息，并将控制命令下发给 STM32 终端。
## 在 EdgeAIoT 系统中的位置
定位：通信层 / 数据中枢。
上游连接 Orange Pi + STM32 边缘 AIoT 管控系统和 MQTT Broker，下游通过串口连接 STM32 环境感知与执行控制终端。
主要数据链路：
STM32 终端 → UART → Linux 网关 → MQTT → Orange Pi 总控层。
主要控制链路：
Orange Pi / MQTT → Linux 网关 → UART → STM32 终端。

## 数据流与控制流
### 设备数据上传流程
1. STM32 采集传感器数据；
2. STM32 通过 UART 发送设备数据；
3. Linux 网关读取串口数据；
4. 网关对数据进行校验、解析和格式转换；
5. 网关将数据封装为 JSON；
6. 网关通过 MQTT 发布给上层系统；
7. Orange Pi 总控层接收并展示或进一步处理数据。
### 控制命令下发流程
1. Orange Pi Agent 或其他上层应用生成控制命令；
2. 控制命令通过 MQTT 发送给 Linux 网关；
3. 网关解析 MQTT 消息；
4. 网关将命令转换为 STM32 可以识别的串口协议；
5. 网关通过 UART 将命令下发给 STM32；
6. STM32 执行控制动作并返回执行结果。
当前控制链路仍需要在后续版本中继续实现和验证。
## 与 STM32 环境感知与执行控制终端的关系
STM32 终端负责传感器采集、状态显示、模式切换、异常报警和执行器控制。
Linux 网关负责接收 STM32 数据、解析通信协议、转发 MQTT 消息，并把上层控制命令下发给 STM32。
两者通过 UART 通信，STM32 不直接承担 MQTT 数据中枢职责。
## 与 Orange Pi + STM32 边缘 AIoT 管控系统的关系
Linux 网关为 Orange Pi 总控层提供统一的设备数据通道。
Orange Pi 负责语音识别、Agent 指令解析、场景联动、知识库调用和状态展示；Linux 网关负责把这些上层指令可靠地传递给 STM32 终端。
## 异常处理
当前已经学习或测试的异常情况包括：
* 串口设备路径不存在；
* 配置文件路径或配置项错误；
* 串口打开失败；
* 接收到的数据格式不符合要求；
* MQTT 连接或消息发送失败；
* Python 依赖未安装或虚拟环境未激活；
* 非法控制命令处理；
后续需要继续补充：
* MQTT 自动重连；
* 串口断开后的恢复机制；
* 数据校验与丢包处理；
* 多线程异常退出和资源释放
